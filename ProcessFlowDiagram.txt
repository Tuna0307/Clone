Process Flow Diagram for iam_log_intelligence_agent_hybridChunking2.py (modular pipeline).

Production workflows (user-selected mode in app.py / CLI --mode):

  A) api_request (default)
     File discovery → schema detection → query-window validation →
     extract_api_request_docs_deterministic → extract_global_evidence_profile →
     select_evidence_chunks → per-file Map LLM → consolidate_reports → PDF

  B) server_monitoring (opt-in)
     File discovery → schema detection → query-window validation →
     load_server_metrics_into_duckdb → LangGraph structured SQL workflow
     (initialize → high_volume_diagnostic → red_herring_filter → evidence_gathering →
      critic → report_synthesis → optional ticket_refinement → finalize) →
     consolidate_reports → PDF

Legacy hybrid server-monitoring chunking, pre-embedding dedup/downselect, signal
filtering, embedding, FAISS kNN, and z-score anomaly scoring have been removed.
Server monitoring no longer produces outputs/faiss/index.faiss artifacts.

-----------------------------------------------------------------------
Entry Point
-----------------------------------------------------------------------

Box 1: Program Start (__main__ or Streamlit app.py)
  CLI: python iam_log_intelligence_agent_hybridChunking2.py [--mode api_request|server_monitoring] <paths>
  UI: streamlit run app.py

-----------------------------------------------------------------------
Common Prefix (both modes)
-----------------------------------------------------------------------

Box 2: File Discovery (pipeline/files.py:get_log_files_from_path)
  Accepts files or directories; collects .log .txt .out .err .msg recursively.

Box 3: Analysis Mode Selection (user-controlled)
  UI sidebar radio or CLI --mode chooses api_request or server_monitoring.
  No query-text routing or structural override.

Box 4: Read Sample Lines
  First ~1200 lines for schema detection via stream_file_lines().

Box 5: Schema Detection (pipeline/parsing.py:detect_log_structure)
  Optional hybrid fallback via schema.detect_log_structure_hybrid when confidence is low.

Box 6: Query Window Validation (pipeline/query.py)
  compute_file_time_coverage + validate_query_window before extraction/analysis.

-----------------------------------------------------------------------
API Request Path (mode="api_request")
-----------------------------------------------------------------------

Box 7: Deterministic API Extraction (pipeline/chunking.py)
  extract_api_request_docs_deterministic groups request lifecycles by thread and
  boundary markers; emits api_signal_event lines for isolated critical signals.
  Deterministic scoring (IAM/error/boundary boosts) — no embeddings or FAISS.

Box 8: File-Wide Evidence Profile (pipeline/dedup.py)
  extract_global_evidence_profile scans the file for aggregate statistics.

Box 9: Evidence Selection (pipeline/evidence.py:select_evidence_chunks)
  Ranked seeds + temporal neighbours + citation IDs + MAP_EVIDENCE_BUDGET_CHARS cap.

Box 10: Map LLM Analysis (pipeline/analysis.py)
  Structured 3-section per-file forensic report using [REF_...] citations.

-----------------------------------------------------------------------
Server Monitoring Path (mode="server_monitoring")
-----------------------------------------------------------------------

Box 11: DuckDB Load (pipeline/server_metrics.py)
  Streams metrics + log_events into in-memory DuckDB with optional time-window filter.

Box 12: Structured SQL Workflow (pipeline/server_sql_graph.py)
  Explicit-phase LangGraph orchestration with mandatory early high-volume diagnostics,
  critic review, and time-series report synthesis. No chunking or FAISS.

-----------------------------------------------------------------------
Reduce + Artifacts
-----------------------------------------------------------------------

Box 13: consolidate_reports (pipeline/reporting.py)
  Cross-file synthesis into the standard 4-section report; citation replacement for API path.

Box 14: PDF Export + AnalysisContext (app.py / followup/context.py)
  outputs/reports/IAM_Forensic_Report.pdf
  outputs/debug/debug_evidence_<file>.txt
  API path sets faiss_index_dir metadata field but does not write FAISS indexes.

-----------------------------------------------------------------------
Follow-up Chat (artifact-first, no full re-pipeline)
-----------------------------------------------------------------------

Box 15: server_monitoring follow-up → followup/server_sql.py (DuckDB SQL)
Box 16: api_request follow-up → followup/answer.py (_debug_ref_candidates from debug evidence)
  Raw log windows remain available via followup/sources.py for future expansion.
